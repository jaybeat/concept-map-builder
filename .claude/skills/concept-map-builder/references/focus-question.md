# 从主题到焦点问题(Claude Code 版)

一个主题(如"数据结构")是一个大领域,塞不进一个焦点问题。这里讲怎么把它收窄成一个(或一组)能用一张图答完的焦点问题。

## 核心区别

主题是名词,焦点问题是问句,而且**一张图只对应一个焦点问题**。面对大主题,任务不是"找出那唯一的问题",而是把它拆成若干各自能一张图答完的焦点问题,必要时挑一个先画。

## 怎么生成候选问题

先想用户的**目的**:学完想能做什么(写代码时选对工具?看懂材料?应付考试?)。目的不同,该问的问题不同;目的也常自带一个统领全局的顶层问题。

再用几把"提问镜头"把主题撬成**关系问题**(避开单纯的"……是什么"):
- 怎么运作(how):X 是如何做到 Y 的?
- 为什么(why):X 为什么会发生 / 为什么这么设计?
- 如何取舍(compare):A 和 B 各适合什么场景?
- 由什么决定(when/what):是什么决定了该选哪个 / 何时用 X?

## 好焦点问题的三条检验

1. 问的是**关系**而非定义(能引出一串命题,而非扁平清单)。
2. **范围一张图答得完**(约 15–25 个概念)。
3. 与**目的对齐**,答完后用户确实能做想做的事;具体到能判断"答完了没"。

## 本 skill 的默认行为(纯文本提问)

Claude Code 没有 `ask_user_input_v0` 这类选项 UI,所以:
- 若用户给的已是一个像样的焦点问题,直接用,进入六步。
- 若给的是大主题或不像问句,用上面的镜头生成 **2–4 个**候选焦点问题,**在回复里编号列出**,简述每个聚焦哪一点,然后**停下,等用户回复选哪个(或另给)**。不要替用户硬选——选错方向,整张图就歪了。

示例(对"数据结构",直接在终端里这样问):
```
这是个大主题,先定焦点。想聚焦哪一个?(回复编号,或自己给一个)
1. 如何为一个问题选对数据结构?(顶层/取舍)
2. 哈希表如何实现平均 O(1) 查找?(机制)
3. 数组与链表各适合什么场景?(取舍)
4. 二叉搜索树为何退化、平衡树如何修正?(因果)
```

## 何时拆成多张互链子图

若用户想覆盖整个大领域:定目的 → 切子领域 → 每块撬成一个焦点问题(各一张图)→ 子图之间靠**共享概念的交叉连接**挂起来(如"时间复杂度"会在多张图复现),其中可有一张统领全局的总览图。

判断信号:当一张图为了"一一对应"而需要的精确连线超过约五六条、线开始打架时,就把某一块拆出去单独成图,而不是继续在一张里加线。
